## Dashboard

The dashboard is _your page_ and allows full customization. Cards are provided for each service (LMS, Timetable, etc.) along with general purpose and custom cards.

Customization is done through the editor.

Quick tips:

- Keep as little pages as possible, to reduce clutter.
- Use the "Global Options" button to set a theme across all tiles on the page at once.
- Note that new tiles don't automatically recieve the global options, you have to resave the global options!
- Customize your theme and styles in the Themes page in settings.

### Tiles and Panels

For a full description of each, go [here](/docs/users/features#dashboard-panels-and-tiles)

Here's a quick overview:

**Tiles**

- Can be dragged anywhere on the page, and snap to the grid.
- Can be resized in both dimensions.
- Have individual options that can be set in the tile editor, such as top bar, etc.

**Panels**

- Snap to the left, top, bottom or right of the main editor.
- Automatically resize to fit the space available.
- Have individual options that can be set in the tile editor, such as top bar, etc.

#### **Tiles**

- **Timetable** category
    - Tiles that show your timetable for the week or day.
- **LMS** category
    - Tiles that show your tasks, modules, etc. relating to your school/LMS.
- **Information** category
    - Tiles that show you general information and stats.

#### **Panels**

- **Navigation panel**
    - Contains your dashboard pages.

## Customization

In the settings page (and onboarding for new users), you can select your theme and further deep customization.

schoolm8 contains a visual editor, however the visual editor may oversimplify and exclude some options. For more control, you can dive into the code option and edit the raw tailwind classes. I'd recommend using an LLM for this, however keep a save of your original settings just in case! LLMs are unpredictable.

### Community themes

<Callout type="warning">
    User generated content is available using this feature. If you wish to disable UGC, you can do
    so in settings, as described [here](/docs/users/settings#ugc-settings)
</Callout>

Community themes are useful for when you want to tweak a single thing in the UI, or go full overboard on your customization, letting fans like you do the work for you!

Community themes are available in the onboarding or Themes settings page.

You can create a theme by posting your current styles, via the Community page under Themes in settings. A more in detailed guide to community themes is linked [here](/docs/users/features#community-themes)

### Code editor

The code editor is a powerful tool, customizing raw classes and theme data, bypassing the limitations of the visual editor in a more efficient manner.

Don't like code? Use an LLM or a community theme! The code editor is for power users who want to customize every little detail, and are comfortable with code. It's also useful for debugging and troubleshooting issues with your theme.

Like code? View more info on customizations and the code editor [here](/docs/developers/customization#code-editor)
